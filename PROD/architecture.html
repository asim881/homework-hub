Key Data Flows
1
Homework Generation
  • Kid selects their name and clicks a day tab
  • App checks localStorage — if assignment already cached, skip API call
  • If not cached: POST to Apps Script with a prompt (kid name, grade, subject, week number)
  • Apps Script POSTs to Anthropic /v1/messages with secret API key in header
  • Claude Haiku returns 5 questions as structured JSON
  • Apps Script relays response back to browser
  • App caches in localStorage and displays questions to kid
2
Answer Grading
  • Kid types answers and clicks Submit
  • App bundles all 5 questions + kid's answers into a grading prompt
  • POST to Apps Script (no action field = Claude call)
  • Apps Script POSTs to Anthropic API — asks Claude to score each answer and give feedback
  • Response includes per-question score + feedback text
  • App displays results screen with total score and feedback per question
  • Grading result immediately triggers the Progress Sync flow below
3
Progress Sync to Sheets
  • After grading, app builds a saveProgress payload (kid, week, day, subject, score, all questions)
  • Payload added to hwSyncQueue in localStorage (ensures no data loss if offline)
  • POST to Apps Script with action: "saveProgress"
  • Apps Script writes one row to Overview tab (summary)
  • Apps Script writes one row per question to kid's own tab
  • If all 7 days done for the week, weekly summary row auto-inserted at top
  • On success, item removed from sync queue
4
Parent Dashboard Load
  • Parent opens parent-dashboard.html in browser
  • POST to Apps Script with action: "getData"
  • Apps Script reads all rows from Overview tab in Sheets
  • Returns array of row objects as JSON
  • Dashboard filters out weekly summary rows, finds current week
  • Renders kid summary cards (current week %, day chips) + full history table
  • Auto-refreshes every 2 minutes — no manual reload needed
Why Each Component — Design Decisions
Why Google Apps Script?
The most important architectural decision
API keys can't be stored safely in browser HTML — anyone can View Source
Apps Script runs server-side on Google's infrastructure — API key never leaves the server
Free with a Google account — no AWS/Heroku/VPS needed
Has native SpreadsheetApp access — writing to Sheets requires no extra OAuth setup
Deploys in minutes as a public HTTPS endpoint
Alternative (direct browser API calls) would expose the key in HTML source
Alternative (Node.js server) would need hosting, deployment, and maintenance
Why Google Sheets?
Progress storage and parent visibility
Zero setup for parents — they already use Google Sheets
Shareable with wife via standard Google sharing — no app login needed
Apps Script can write to Sheets natively without extra API auth
Free, no database to manage or back up
Structured enough for the parent dashboard to query
Alternative (Firestore/Supabase) would need separate auth and cost money
Why a Single HTML File?
Simplicity over frameworks
No npm, no build tools — edit and push, GitHub Pages serves it instantly
Works on any Chromebook out of the box
Easy to share, inspect, and modify in future
DEV/ for local testing, PROD/ for deployment — simple two-folder workflow
Alternative (React/Next.js app) would need a build pipeline and more complexity
Why Claude Haiku?
AI model selection
Fastest and cheapest Claude model — suitable for real-time homework sessions
Smart enough to generate age-appropriate questions and grade answers accurately
Questions vary every week — kids can't memorise answers
Per-question feedback teaches kids why they got something wrong
Alternative (GPT-4) costs more and is slower for this use case
Alternative (static question banks) require manual curation and don't adapt
Why localStorage?
Client-side state management
Instant reads and writes — no network call needed to check if homework is done
Assignment caching avoids duplicate AI calls (and cost) if kid revisits a tab
Sync queue ensures progress isn't lost if Sheets write fails
Limitation: per-device only — switching Chromebooks loses local progress
Backlog item: cross-device sync via Sheets session state
Why GitHub Pages?
Hosting — replaced Google Sites May 2026
Free, stable URL (asim881.github.io) — add once to Family Link, works forever
Deploys automatically on every git push — no manual copy-paste
Full version history via git — roll back any change instantly
Kids access via bookmark, works on any Chromebook
Google Sites discarded: its embed domain got a new random numeric prefix on every browser tab, making Family Link whitelisting impossible
